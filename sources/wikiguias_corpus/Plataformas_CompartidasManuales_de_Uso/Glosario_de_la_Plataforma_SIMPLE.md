[¶](#glosario-de-términos-de-la-plataforma-simple) Glosario de términos de la plataforma SIMPLE
-----------------------------------------------------------------------------------------------

### [¶](#a) A

* Acciones: Conjunto de acciones que permiten la ejecución de distintas funcionalidades dentro de un proceso, tales como, enviar correo, generar variables, consultar servicios web, etc.
* Acompañamiento equipo de digitalización: Servicio que entrega la Secretaría de Gobierno Digital a través de su equipo de digitalización en el desarrollo de los procesos institucionales cuyas actividades y requisitos se encuentran en la guía de servicio de acompañamiento.
* Ambiente de desarrollo: Es el ambiente compartido para el desarrollo de nuevos procesos de los distintos órganos de la administración del Estado (OAE), que utilizan SIMPLE.
* Ambiente de Capacitación: Es el ambiente compartido para el desarrollo de los manuales de capacitación y ejecución del taller aprendiendo SIMPLE.
* Ambiente productivo: Es el ambiente entregado a cada órgano de la administración del Estado (OAE), por la Secretaría de Gobierno Digital de manera gratuita para disponibilizar sus procesos de cara a las personas.
* API (Application Programming Interface): conjunto de reglas, protocolos y herramientas que permite que diferentes aplicaciones de software se comuniquen entre sí. En términos simples, es una interfaz que define cómo un sistema puede interactuar con otro para enviar o recibir datos.
* Aprendiendo SIMPLE: Taller práctico desarrollado por el equipo de digitalización que forma parte de la fase de capacitación de cada funcionario que trabajará con SIMPLE.
* Auditoría de Procesos: Revisión y análisis de los flujos de trabajo para garantizar el cumplimiento de políticas y normativas.
* Automatización de Procesos: Uso de tecnología para realizar tareas y procesos con mínima intervención humana.

### [¶](#b) B

* Back-end: Es la parte del software que se encarga de manejar la lógica, el procesamiento de datos y la interacción con el servidor. Es el "lado del servidor" de una aplicación, que no es visible para el usuario final pero es esencial para que el sistema funcione correctamente. En SIMPLE corresponde a la sección dónde se encuentra por ejemplo el modelador de procesos.
* BPMN (Business Process Model and Notation): Lenguaje estándar para modelar procesos de negocio que utiliza diagramas de flujo.
* BPMS (Business Process Management System): Plataforma tecnológica que permite modelar, automatizar, monitorear y optimizar procesos de negocio en una organización.
* Buenas prácticas de digitalización: Conjunto de recomendaciones entregadas a los equipos institucionales a través de la guía de Buenas prácticas de digitalización que es necesario considerar para la digitalización de un proceso.

### [¶](#c) C

* Campo botón asíncrono: Campo permite ejecutar acciones en un mismo formulario sin necesidad de presionar el botón siguiente.
* Campo botón siguiente: Campo que permite reemplazar el botón siguiente proporcionado por defecto.
* Campo checkbox: Campo que permite agregar varias opciones de selección y en la que se puede marcar más de una.
* Campo comunas: Campo que despliega las regiones y comunas del país para ser seleccionado en un formulario.
* Campo date: Campo que permite seleccionar una fecha del calendario y que puede ser restringida de acuerdo a las necesidades del órgano de la administración del Estado (OAE).
* Campo documento: Campo que permite hacer referencia a un documento creado en la plataforma.
* Campo file: Campo que permite subir archivos a la plataforma con un máximo de 15 MB.
* Campo file transfer: Campo que te permite subir archivos a la plataforma sin importar el tamaño pero estos archivos solo cuentan una vigencia de 30 días desde el momento que se carga un archivo.
* Campo grilla: Campo que permite agregar una tabla editable, la cual al momento de agregarla nos permite agregar las columnas que queramos que tenga la tabla.
* Campo grilla de datos externos:
* Campo instituciones: Campo que despliega el nombre de los diferentes órganos de la administración del Estado de Chile.
* Campo Javascript: Campo que permite agregar código Javascript a tu proceso.
* Campo moneda: Campo que despliega un listado de las monedas utilizadas en diferentes países del mundo.
* Campo países: Campo que despliega un listado de los países del mundo.
* Campo párrafo: Campo que permite utilizar un mayor número de caracteres y que puede ser configurado utilizando lenguaje HTML.
* Campo password: Campo que permite agregar una etiqueta html input de tipo “password”.
* Campo provincias: Campo que despliega un listado de las regiones, provincias y comunas de Chile.
* Campo radio: Campo que permite agregar varias opciones de selección pero sólo se puede marcar una.
* Campo Select: Campo que permite agregar varias opciones a través de una lista desplegable.
* Campo subtítulo: Campo que te permite agregar un texto con la etiqueta html “H4” por defecto, pero también puede ser configurado para utilizar otras etiquetas HTML, como por ejemplo: “H1”, “H2”, etc.
* Campo textarea: El campo textarea se utiliza para crear un área de texto multilínea. Es un componente ampliamente utilizado para permitir la entrada de texto por parte del usuario, como comentarios, descripciones, o cualquier otra forma de texto que necesite más de una línea.
* Campo textbox: El campo textbox se utiliza para crear un área de texto básica de una sola línea.
* Campo título: Campo que te permite agregar un texto con la etiqueta html “H3” por defecto, pero también puede ser configurado para utilizar otras etiquetas HTML, como por ejemplo: “H1”, “H2”, etc.
* Ciclo de Vida de Procesos: Fases de un proceso en un BPMS: modelado, implementación, ejecución, monitoreo y optimización.
* Consultar rest (acción): Acción que permite configurar toda la información necesaria para consumir un servicio web de tipo rest, además de configurar el nombre de la variable donde se almacenará la respuesta del servicio.
* Consultar soap (acción): Acción que permite configurar toda la información necesaria para consumir un servicio web de tipo soap, además de configurar el nombre de la variable donde se almacenará la respuesta del servicio.
* Crear conexión de unión: Conexión necesaria cuando se usa la conexión paralela ya que esta nos permitirá pasar a la siguiente tarea, al completar todas las tareas paralelas.
* Crear conexión paralela: Conexión que permite ejecutar más de una tarea al mismo tiempo en un mismo proceso.
* Crear conexión paralela por evaluación: Conexión que permite ejecutar más de una tarea al mismo tiempo en un mismo proceso dependiendo si se cumple o no la condición.
* Crear conexión por evaluación: Conexión que a través de una condición nos permite avanzar a una siguiente tarea u otra.
* Crear conexión secuencial: Conexión que nos marca cual es la siguiente tarea a ejecutar.
* Crear tarea: Permite generar un componente llamado tarea, la cual contiene los pasos, acciones y configuraciones necesarias para la ejecución del proceso.
* Cuenta: Es la creación en la nube de la institución usuaria de SIMPLE, la que previamente deberá haber nombrado a un administrador que será la contraparte del Equipo de Digitalización y el responsable de administrar y gestionar todo lo relacionado a la cuenta.

### [¶](#d) D

* Descargar documento (acción): Acción que permite descargar un documento que esté almacenado en una url pública, y transformarlo en base64 dentro de una variable en simple.
* Diagrama de proceso: Representación visual de las etapas, pasos y decisiones involucradas en un flujo de trabajo o proceso. Se utiliza para analizar, documentar, modelar o mejorar procesos en diversas áreas. Este tipo de diagrama ayuda a comprender cómo se ejecuta un proceso y dónde se pueden optimizar o resolver problemas.
* Digitalización de Procesos: Transformación de procesos manuales o tradicionales en flujos automatizados mediante herramientas tecnológicas.
* Diseñador: Módulo del backend de SIMPLE dónde un digitalizador crea y configura las tareas de un proceso.
* Diseño/rediseño de procesos: Crear o modificar procesos para hacerlos más eficientes, alineados con los objetivos de la organización y las necesidades de las personas.
* Documentos: Archivos de texto que pueden ser generados a través de un formato HTML dentro de la plataforma simple

### [¶](#e) E

* Enviar correo (acción): Acción que permite configurar el “Remitente”, “Copia”, “Copia Oculta”, “Asunto”, el Contenido del correo electrónico y los documentos adjuntos que contendrá el correo electrónico.
* Enviar documento (acción): Acción que permite mediante una llamada rest, enviar un documento generado en simple a un endpoint disponibilizado por la institución. Este documento se envía en formato base64
* Equipo de digitalización: Equipo de la Secretaría de Gobierno Digital que brinda herramientas y asesoría para capacitar a los órganos de la administración del Estado en el modelado y optimización de procesos, facilitando su digitalización para ofrecer servicios más eficientes y de mayor calidad a los ciudadanos.

### [¶](#f) F

* Formularios: Es un lienzo en el cual se puede agregar todos los campos de SIMPLE.
* Front-end: Es la parte visible e interactiva de una aplicación o sitio web con la que las personas interactúan. En SIMPLE quienes interactúan a través del front-end son los funcionarios encargados que interactúan en un proceso o las personas usuarias.

### [¶](#g) G

* Generar documento (acción): acción que permite generar un documento dentro del trámite que se está ejecutando. El documento debe estar previamente elaborado en la sección “documentos”
* Generar variable (acción): Acción que permite generar un nueva variable dentro del trámite que se está ejecutando.
* Gestión del Cambio: Estrategia para asegurar la adopción efectiva de los procesos digitalizados en la organización.
* Gestión de procesos: Disciplina y práctica de identificar, diseñar, documentar, implementar, monitorear y optimizar los procesos de negocio dentro de una organización para alcanzar sus objetivos estratégicos.
* Guía de uso de SIMPLE SaaS: Guía que contiene los elementos necesarios para utilizar SIMPLE. Requisitos y compromisos que se deben cumplir.

### [¶](#i) I

* Integración: Conexión de sistemas, aplicaciones y servicios externos al BPMS para intercambiar información y ejecutar tareas.

### [¶](#k) K

* KPI (Key Performance Indicator): Métricas clave para evaluar el desempeño y eficiencia de los procesos de negocio.

### [¶](#m) M

* Manuales de digitalización: Manuales desarrollados por el Equipo de Digitalización para ser desarrollados por los digitalizadores y aprender acerca de la digitalización con SIMPLE.
* Modelado de Procesos: Representación gráfica de un proceso, comúnmente usando estándares como BPMN (Business Process Model and Notation).
* Monitorización de Procesos: Seguimiento en tiempo real del desempeño de un proceso para identificar cuellos de botella o ineficiencias.

### [¶](#o) O

* Optimización de Procesos: Mejora continua de los flujos de trabajo para aumentar su eficiencia y reducir costos.

### [¶](#p) P

* Presentación de procesos: Reunión que sostienen los digitalizadores de un órgano de la administración del Estado - posterior al acompañamiento- en el que presentan el proceso desarrollado a todos los integrantes del equipo de digitalización.
* Procesos End-to-End: Procesos completos que abarcan todas las etapas, desde el inicio hasta la entrega del resultado final.

### [¶](#r) R

* Redirección (acción): Acción que permite redireccionar al usuario, a una url externa a SIMPLE, luego de completar una tarea o algún paso dentro del trámite.
* Reglas de Negocio: Directrices o condiciones que determinan las acciones y decisiones dentro de un flujo de proceso.

### [¶](#s) S

* SaaS (Software as a service): Modelo de software en el que las aplicaciones se alojan en la nube y se ofrecen a los usuarios a través de Internet.
* Simple Day: Webinar en el que los participantes tienen la oportunidad de conocer más acerca del modelador de procesos SIMPLE, sus usos, el acompañamiento y cómo acceder a él.
* SIMPLE: Sistema de Implementación de Procesos Ligeramente Estandarizados. Es un sistema modelador de procesos también conocido como BPMS.

### [¶](#t) T

* Tareas de SIMPLE: Actividad que se configura para el modelado de procesos.
* Task: Unidad mínima de trabajo dentro de un flujo de proceso en un BPMS.

### [¶](#v) V

* Variable: Es un dato o valor que se encuentra almacenado en la plataforma para un trámite específico.

### [¶](#w) W

* Workflow: Secuencia automatizada de tareas, reglas y decisiones que definen cómo se ejecuta un proceso.
